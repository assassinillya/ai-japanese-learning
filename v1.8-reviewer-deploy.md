# v1.8 Reviewer 补充部署方案

本文档用于把 `ai-japanese-learning v1.8` 部署到一台没有本地开发环境、但已经安装 Docker 的 Windows 电脑，并通过内网穿透提供外部访问。

## 目标架构

```text
外部访问地址
  -> 内网穿透客户端
  -> Windows 主机 localhost:8080
  -> Docker Compose app 容器
  -> Docker Compose postgres 容器
```

目标 Windows 电脑不需要安装 Go、Node.js 或 PostgreSQL。运行时依赖由 Docker Compose 提供。

## 前置条件

- Windows 电脑已安装 Docker Desktop，并确认 Docker Engine 正常运行。
- 可以访问 GitHub 下载项目代码。
- 已准备一个内网穿透方案，例如 frp、cloudflared、ngrok 或 cpolar。
- 如果要使用真实 AI，需要准备对应 provider 的 API Key；否则可以先用 `placeholder` 跑通流程。

## 推荐目录

建议把项目放在固定目录，例如：

```powershell
C:\deploy\ai-japanese-learning
```

如果目标机没有 Git，可以在 GitHub 下载 zip 后解压到该目录。

## 获取代码

有 Git 时：

```powershell
mkdir C:\deploy
cd C:\deploy
git clone https://github.com/assassinillya/ai-japanese-learning.git
cd ai-japanese-learning
```

没有 Git 时：

1. 打开 GitHub 项目页面。
2. 下载当前 `main` 分支 zip。
3. 解压到 `C:\deploy\ai-japanese-learning`。
4. 在 PowerShell 中进入该目录。

## 配置环境变量

在项目根目录创建 `.env`：

```env
POSTGRES_PASSWORD=replace-with-strong-password
POSTGRES_PORT=5432
APP_PORT=8080
APP_TOKEN_SECRET=replace-with-random-long-secret

AI_PROVIDER=placeholder
AI_PROVIDER_NAME=
AI_BASE_URL=https://api.openai.com/v1
AI_API_KEY=
AI_MODEL=gpt-4o-mini
AI_API_VERSION=
```

说明：

- `POSTGRES_PASSWORD`：数据库密码，部署时必须改掉默认值。
- `APP_TOKEN_SECRET`：登录 token 签名密钥，部署时必须改成随机长字符串。
- `APP_PORT`：宿主机暴露端口，内网穿透应指向 `127.0.0.1:8080`。
- `AI_PROVIDER=placeholder` 可以不配置真实 AI，先验证部署链路。

如果要启用真实 AI，可以改成对应 provider，例如：

```env
AI_PROVIDER=openai
AI_BASE_URL=https://api.openai.com/v1
AI_API_KEY=sk-...
AI_MODEL=gpt-4o-mini
```

## 启动服务

在项目根目录执行：

```powershell
docker compose up -d --build
```

查看容器状态：

```powershell
docker compose ps
```

查看日志：

```powershell
docker compose logs -f app
```

本机访问：

```text
http://localhost:8080
```

## 初始化数据库

当前项目没有独立的 migration runner，首次部署需要手动执行 migrations。

在项目根目录依次执行：

```powershell
Get-Content migrations\001_init.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\002_articles_v02.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\003_challenge_reading_v05.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\004_challenge_metadata_v05_fix.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\004_user_ai_configs_and_dictionary_forms.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\005_post_reading_quiz_v06.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\006_vocabulary_review_v07.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\007_ai_cache_logs_v08.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
Get-Content migrations\008_vocabulary_review_three_questions.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
```

可选：导入 MVP 种子数据：

```powershell
Get-Content seeds\002_mvp_seed_v09.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
```

如果重复执行 migration 时出现对象已存在或约束已存在，需要根据错误确认是否已经执行过。大多数 `CREATE TABLE IF NOT EXISTS` 和 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 可重复执行，但个别历史迁移仍建议只执行一次。

## 内网穿透

内网穿透目标统一设置为：

```text
127.0.0.1:8080
```

### 方案 A：frp

适合有自有服务器或固定公网入口的部署。

`frpc.ini` 示例：

```ini
[common]
server_addr = your-frp-server.example.com
server_port = 7000
token = replace-with-frp-token

[ai-japanese-learning]
type = http
local_ip = 127.0.0.1
local_port = 8080
custom_domains = japanese.example.com
```

启动 frpc 后，通过 `https://japanese.example.com` 或 frp 服务端配置的地址访问。

### 方案 B：cloudflared tunnel

适合有 Cloudflare 域名的部署。目标服务配置为：

```text
http://127.0.0.1:8080
```

优点是不需要在路由器开放端口，也比较适合长期运行。

### 方案 C：ngrok / cpolar

适合临时演示或低成本试用。创建 HTTP 隧道，目标地址填：

```text
http://127.0.0.1:8080
```

## 日常运维命令

停止服务：

```powershell
docker compose down
```

重启服务：

```powershell
docker compose restart
```

更新代码后重新构建：

```powershell
git pull
docker compose up -d --build
```

查看 app 日志：

```powershell
docker compose logs -f app
```

查看数据库日志：

```powershell
docker compose logs -f postgres
```

备份数据库：

```powershell
docker compose exec -T postgres pg_dump -U postgres japanese_learning > japanese_learning_backup.sql
```

恢复数据库：

```powershell
Get-Content japanese_learning_backup.sql | docker compose exec -T postgres psql -U postgres -d japanese_learning
```

## 注意事项

- GitHub `main` 只包含已经提交并 push 的代码；本地未提交改动不会出现在目标 Windows 电脑上。
- `docker-compose.yml` 会创建 `postgres_data` volume，数据库数据会保存在 Docker volume 中。不要随意删除 volume。
- 第一次部署后建议立刻注册管理员或测试账号，并确认登录、上传文章、查词、生词本和复习页都可访问。
- 如果使用真实 AI，建议先在个人中心保存用户级 AI 配置，再验证查词和题目生成。
- 内网穿透公网地址如果给多人使用，必须使用强 `APP_TOKEN_SECRET`、强数据库密码，并避免暴露 PostgreSQL 端口到公网。
